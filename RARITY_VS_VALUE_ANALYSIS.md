# 희귀도 vs 가치 분석 보고서

## 현재 상황 분석

### 1. 희귀도 (Rarity)
- **정의**: AI가 분석한 카드의 고유 등급 (1~5성)
- **출처**: Gemini AI가 카드 이미지를 분석하여 직접 제공
- **용도**:
  - XP 계산 (희귀도 × 10)
  - 가격 계산의 기본값
  - 별(★) 표시로 시각화
  - 필터링 및 정렬

### 2. 가치 (Value/Price)
- **정의**: 희귀도를 기반으로 HP와 파워레벨을 고려하여 계산된 가격
- **계산식**: `basePrice[rarity] × hpMultiplier × powerMultiplier`
- **표시**: 가치 등급으로 변환 (전설, 초희귀, 희귀, 보통, 일반)
- **용도**: 필터링 및 정렬

## 문제점

### 1. 개념 중복
- **희귀도**: 1~5성 (별 표시)
- **가치 등급**: 전설(5), 초희귀(4), 희귀(3), 보통(2), 일반(1)
- 두 개념이 거의 동일한 의미로 사용됨

### 2. 계산 방식
```javascript
// 가치는 희귀도를 기반으로 계산
basePrices = { 1: 500, 2: 2000, 3: 5000, 4: 15000, 5: 50000 }
estimatedPrice = basePrice[rarity] × hpMultiplier × powerMultiplier
```
- 가치가 희귀도에 크게 의존
- HP와 파워레벨의 영향이 제한적
- 결과적으로 희귀도와 거의 일치

### 3. 아이들에게 혼란
- "희귀도 5성" vs "가치 전설" - 같은 의미인데 다른 표현
- 두 개의 필터가 비슷한 결과를 보여줌
- 어떤 것을 기준으로 해야 할지 모호함

### 4. UI 중복
- 희귀도 필터: 1~5성별 필터
- 가치 필터: 보물/좋은 것/일반 필터
- 두 필터가 거의 같은 카드를 보여줌

## 개선 방안

### 방안 1: 희귀도 중심 (추천) ⭐
**장점**:
- 단순하고 명확함
- AI가 직접 분석한 정보 활용
- 아이들이 이해하기 쉬움
- 게임적 요소 (별 등급)

**변경사항**:
- 가치 필터 제거
- 가치 등급 표시 제거
- 희귀도만으로 필터링 및 정렬
- 희귀도에 따른 시각적 표현 강화

### 방안 2: 가치 중심
**장점**:
- 실제 가치를 반영
- HP와 파워레벨도 고려

**단점**:
- 계산이 복잡함
- 희귀도와 거의 동일한 결과
- 아이들이 이해하기 어려울 수 있음

### 방안 3: 두 개념 명확히 구분
**희귀도**: 카드의 등급 (1~5성)
**가치**: 실제 시장 가치 (전설, 초희귀 등)

**문제점**:
- 현재는 두 개념이 거의 일치
- 구분이 모호함
- 아이들에게 여전히 혼란
