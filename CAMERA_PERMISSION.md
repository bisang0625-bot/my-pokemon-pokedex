# 카메라 접근 권한 허용 가이드

웹 브라우저에서 카메라를 사용하려면 사용자의 명시적인 권한이 필요합니다. 아래는 주요 브라우저별 권한 허용 방법입니다.

## 주요 브라우저별 권한 허용 방법

### 🌐 Chrome / Edge (Chromium 기반)

1. **권한 요청 팝업이 나타나면:**
   - "허용" 또는 "Allow" 버튼 클릭

2. **이미 차단한 경우:**
   - 주소창 왼쪽의 자물쇠 아이콘(🔒) 또는 정보 아이콘(ℹ️) 클릭
   - "카메라" 또는 "Camera" 항목 찾기
   - "허용" 또는 "Allow" 선택
   - 페이지 새로고침

3. **설정에서 직접 변경:**
   - 주소창에 `chrome://settings/content/camera` 입력
   - 또는 Chrome 설정 → 개인정보 및 보안 → 사이트 설정 → 카메라
   - 차단된 사이트 목록에서 해당 사이트 찾아서 허용

### 🦊 Firefox

1. **권한 요청 팝업이 나타나면:**
   - "허용" 또는 "Allow" 버튼 클릭

2. **이미 차단한 경우:**
   - 주소창 왼쪽의 자물쇠 아이콘(🔒) 클릭
   - "권한" 또는 "Permissions" 클릭
   - "카메라 사용" 또는 "Use Camera" 옆의 드롭다운에서 "허용" 선택

### 🍎 Safari (macOS / iOS)

1. **macOS:**
   - Safari → 환경설정 → 웹사이트 → 카메라
   - 해당 사이트를 찾아서 "허용" 선택

2. **iOS (iPhone/iPad):**
   - 설정 → Safari → 카메라
   - 또는 Safari에서 사이트 방문 시 권한 요청 팝업에서 "허용" 선택

### 📱 모바일 브라우저

#### Android Chrome
- 권한 요청 팝업에서 "허용" 클릭
- 또는 설정 → 앱 → Chrome → 권한 → 카메라 허용

#### iOS Safari
- 권한 요청 팝업에서 "허용" 클릭
- 또는 설정 → Safari → 카메라

## 중요 사항

### 🔒 HTTPS 필요
- 대부분의 브라우저는 **HTTPS 연결**에서만 카메라 접근을 허용합니다
- 개발 중에는 `localhost`가 예외적으로 허용됩니다
- 프로덕션 배포 시에는 반드시 HTTPS를 사용해야 합니다

### 🏠 localhost 예외
- 개발 환경(`http://localhost`)에서는 카메라 접근이 가능합니다
- Vite 개발 서버는 기본적으로 `localhost`에서 실행되므로 문제없이 작동합니다

### 🚫 권한이 차단된 경우
앱에서 다음과 같은 오류 메시지가 표시됩니다:
```
카메라 접근 권한이 필요합니다.
```

이 경우 위의 방법으로 브라우저 설정을 변경한 후 페이지를 새로고침하세요.

## 문제 해결

### 카메라가 작동하지 않는 경우

1. **다른 앱이 카메라를 사용 중인지 확인**
   - 다른 앱에서 카메라를 사용하고 있으면 웹 앱에서 접근할 수 없습니다

2. **브라우저 재시작**
   - 권한 설정을 변경한 후 브라우저를 완전히 종료하고 다시 시작

3. **브라우저 업데이트**
   - 최신 버전의 브라우저를 사용하고 있는지 확인

4. **개발자 도구 확인**
   - F12를 눌러 개발자 도구 열기
   - Console 탭에서 오류 메시지 확인

### 테스트 방법

1. 개발 서버 실행: `npm run dev`
2. 브라우저에서 `http://localhost:5173` 접속
3. "스캔" 페이지로 이동
4. "카메라 시작" 버튼 클릭
5. 권한 요청 팝업에서 "허용" 선택

## 보안 팁

- 신뢰할 수 있는 사이트에서만 카메라 권한을 허용하세요
- 사용하지 않을 때는 브라우저 설정에서 권한을 차단할 수 있습니다
- 카메라 아이콘이 주소창에 표시되면 카메라가 활성화된 상태입니다
